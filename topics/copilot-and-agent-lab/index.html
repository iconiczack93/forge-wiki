    Copilot & Agent Lab

    3 min read · Last updated April 04, 2026 · 579 words

    FORGE's autonomous coding and sourcing research systems.

    Copilot: Persistent Claude Code Pipe

    FORGE Copilot is a fully autonomous Claude Code agent running as a dedicated persistent process with bidirectional stdin/stdout streaming. Unlike traditional API calls, Copilot stays alive between Discord messages—near-instant after first boot.

    Architecture

    • One persistent Claude Code CLI process managed by FORGE
    • Bidirectional JSON streaming: user messages written to stdin, responses streamed from stdout
    • Max subscription OAuth: strips API key, uses stored credentials
    • No claude-agent-sdk: avoids Pydantic V1/V2 conflicts with ChromaDB in the same process
    • Auto-recovery: health monitor (30s interval) restarts dead processes transparently
    • Message queue: concurrent messages queued automatically, no blocking

    Configuration

    SessionFile: data/copilot_session.json (OAuth state)
PIDFile: data/copilot_pid.json (process tracking)
MCPFile: data/copilot_mcp.json (registered MCPs)
ActiveTask: data/copilot_active_task.json (current work)

    Constants: - MAX_MSG_LEN: 1900 chars (Discord limit with margin) - MAX_TURNS: 30 per response - PIPE_IDLE_TIMEOUT: 600s (stop idle process) - HEALTH_CHECK_INTERVAL: 30s - QUEUE_MAX_SIZE: 20 messages

    System Prompt Rules

    Copilot knows: - Full FORGE access (Read, Write, Edit, Bash, Grep, Glob) - Root: C:\Users\User\FORGE\ (absolute paths mandatory) - Main: bot.py (~7300 lines), modules, cogs, config - Critical: concise output (2000 char Discord limit), never echo context - Jump straight to ACTION—first line should be what it's doing, not what it knows - Match Zack's casual, straight-to-point style

    Copilot Awareness

    The copilot_awareness.py module detects when a message needs code changes:

    • Feature requests: "add new command"
    • Bug fixes: "fix the issue with"
    • Behavior changes: "make forge respond"
    • Integration: "hook up X API"
    • Code-specific: "refactor", "rewrite", ".py"

    If a request needs code work, FORGE suggests routing to Copilot automatically.

    Agent Lab: Sourcing Debate System

    Agent Lab is two autonomous AI agents (Scout 🔍 and Analyst 📊) debating sourcing strategies and candidate pipeline approaches. Runs hourly 8am-11pm in the 💡-lab channel.

    Scout vs Analyst

    Role Focus Style
    Scout 🔍 Creative sourcing, unconventional channels Brainstormer, experimenter
    Analyst 📊 Data-driven yield/cost/ROI Evaluator, optimizer

    Session Types

    1. JD Breakdown (📋): Sourcing angles, keywords, Boolean strings, talent communities
    2. Channel Strategy (📡): Compare Slack, GitHub, LinkedIn X-ray, Discord, Reddit, StackOverflow
    3. Outreach Playbook (✉️): Messaging strategies, personalization at scale, timing/tone
    4. Pipeline Review (📊): Bottleneck analysis, conversion rates, source quality
    5. Competitive Sourcing (🏢): Target company strategies, org mapping, signal detection
    6. Wild Card (🤯): Unconventional approaches (commit patterns, blog analysis, patent filings)
    7. VPS Autonomy (🖥️): FORGE 24/7 independence—monitoring, scheduling, self-healing

    Configuration

    • Separate API key: isolates costs from main FORGE usage
    • Default rounds: 6, max 12
    • Daily limit: 30 sessions
    • Cooldown: 30 minutes between sessions
    • Model: Haiku (fast, cost-effective)

    Interactive Mode

    Zack can join a session and steer the agents in real-time. Reply to the agents' messages to influence their analysis.

    "brainstorm 5 outreach playbooks"
→ Scout & Analyst debate 5 different outreach strategies
→ Reply with feedback to steer them

"done" → wraps up the session

    Key Files

    File Purpose
    modules/copilot.py Copilot process manager, health monitor, JSON streaming
    core/copilot_awareness.py Detects code-change requests, auto-routes to Copilot
    core/copilot_context.py Assembles rich context (goals, handoff, decisions, failures)
    modules/agent_lab.py Scout & Analyst session orchestration, batch mode
    cogs/agent_lab.py Discord action handlers (brainstorm, interactive, history)
