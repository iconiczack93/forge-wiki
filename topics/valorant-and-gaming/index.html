    Valorant & Gaming

    FORGE's Valorant API integration, esports data pipeline, and gaming activity tracking.

    Valorant API: Henrik's Unofficial API

    Real-time player stats, rank info, and match history via Henrik Dev API.

    Core Features

    • Player Rank: competitive rank, RR (rating points), ELO, peak rank
    • Last Match: detailed stats (kills, deaths, assists, ACS, headshot %)
    • Match History: multi-match lookup with per-map breakdown
    • Account Info: player profile, region, agent list

    Usage

    from modules.valorant import get_rank, get_last_match, get_match_history

# Initialize with player defaults
init_valorant(api_key="...", default_name="Zack", default_tag="haha1")

# Get rank
rank_info = await get_rank()  # uses defaults
rank_info = await get_rank("Asuna", "main")  # specific player

# Match history (last 5 matches)
history = await get_match_history(count=5)

    API: Async via requests in executor (non-blocking in Discord context).

    VLR.gg Esports Integration

    Professional Valorant esports data: player stats, team schedules, per-map breakdowns. Uses vlrdevapi (free, no auth).

    Data Sources

    • VLR.gg: esports matches, results, player averages
    • PrizePicks API: unauthenticated current projection lines
    • Auto-Grade: match player picks against actual VLR results

    Functions

    Function Purpose
    search_player(name) Find player by IGN, returns player_id + country
    get_player_profile(player_id) Name, team, country, peak rating
    get_player_map_kills(player_id, map_name) Per-map kill count (last N matches)
    get_player_averages(player_id) K/D/A, ACS, kills per round (cached 30min)
    search_matches(date_range) Upcoming/completed matches
    get_pp_lines() Current PrizePicks Valorant prop lines

    Caching

    • Player ID: in-memory cache (frequent lookups)
    • Averages: 30min TTL (avoid re-hitting VLR)
    • Lines: fresh on each call (PrizePicks updates frequently)

    Valorant Lineups Database

    Lineup videos/GIFs for agents on specific maps and sites.

    Agent Coverage

    26 agents with full ability mappings: - Sentinels: Cypher, Killjoy, Chamber - Duelists: Jett, Reyna, Phoenix, Yoru, Neon - Initiators: Sova, Breach, Skye, Fade, Gekko, Kay/O - Controllers: Viper, Omen, Brimstone, Astra, Clove, Iso, Deadlock

    Storage

    SQLite forge.db table: valorant_lineups - Agent, map, site (A/B/mid) - Ability name - Video URL (YouTube/Invidious) - Notes, saved date

    Endpoints

    await search_lineups("sova", "haven", "a site", "recon bolt")
→ returns [{"url": "...", "title": "...", "description": "..."}]

await save_lineup(url, agent, map_name, site, ability, notes="")
await get_saved_lineups(agent, map_name)
await delete_lineup(lineup_id)

    Val Swarm: 11-Agent Predictor

    Tiered analysis for PrizePicks Valorant props. Runs on VPS only (FORGE_INSTANCE == "vps").

    Architecture

    Tier 1 (8 agents, pure math, $0): - Form, Map, Matchup, Role, FirstBlood, Variance, Rounds, OpponentStrength

    Tier 2 (2 agents, Haiku, ~$0.01): - EdgeFinder, DevilsAdvocate

    Tier 3 (1 agent, Haiku, ~$0.01): - Synthesizer (leader, synthesizes all signals)

    Cost & Data

    • Total: ~$0.02-0.03 per full analysis
    • Data: $0 (VLR + PrizePicks free APIs)
    • Output: per-player verdict (over/under recommendation + confidence)

    Usage

    from core.val_swarm import run_swarm_analysis

# Analyze all current lines
results = await run_swarm_analysis()

# Specific players
results = await run_swarm_analysis(players=["starxo", "Asuna"])

    Game Session Tracker

    Monitors Discord presence for gaming activity (Valorant, etc.). Logs to SQLite with weekly summaries.

    Tracking

    • Requires: presences + members privileged intents
    • Trigger: presence update → detect game start/stop
    • Storage: game_sessions table
    • game_name, started_at, ended_at, duration_minutes

    Usage

    await on_presence_update(before_member, after_member)
→ auto-detects Valorant starts/stops

await get_session_summary(days=7)
→ weekly gaming summary

    Discord Integration: Valorant Actions

    The [[valorant_actions.py]] cog handles classifier intents:

    Action Intent Handler
    val_rank "what's my rank" get_rank()
    val_last_match "last game" get_last_match()
    val_history "last 5 matches" get_match_history(count=5)
    val_account "account info" get_account()
    val_lineup "sova lineup haven" search_lineups()
    val_lineup_save "save this lineup" save_lineup()
    val_lineup_library "my lineups" get_saved_lineups()
    val_lineup_delete "delete lineup X" delete_lineup()
