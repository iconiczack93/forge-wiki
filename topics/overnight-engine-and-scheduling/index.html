    Overnight Engine & Scheduling¶

    Overnight productivity engine for FORGE — makes idle VPS hours productive without burning Zack's interactive time.

    Overview¶

    The Overnight Engine is a family of systems that execute deep work (research, profiling, maintenance) when FORGE is idle — typically 9 PM to 7 AM Pacific. Instead of wasting 10 idle hours per night, these systems:

    • Continue pipeline work — score unscoreds, enrich profiles with GitHub data
    • Build company intelligence — research profiles for target companies
    • Raid company rosters — discover candidates via LinkedIn from unscanned AI companies
    • Replenish sourcing lists — auto-discover new companies to raid
    • Dream & synthesize — connect unrelated knowledge, surface insights
    • Maintain systems — re-index knowledge base, prune duplicates, decay stale memories

    Cost cap: $10/night (actual spend typically $3–6)
    Architecture: APScheduler (cron) + 3-model pipeline (Haiku → Opus → Sonnet)
    Runs on: VPS only (7 PM–7 AM PDT)

    Core Systems¶

    1. Overnight Engine (core/overnight_engine.py)¶

    Orchestrator for all overnight work cycles.

    Component Details
    Cycles 4 per night at 9:30 PM, 12:30 AM, 3:30 AM, 6:30 AM Pacific
    Schedule CronTrigger via APScheduler (UTC hours 5, 8, 11, 14 = PDT -7)
    Cost tracking Nightly cap $10 USD, resets each night
    Task priority Company raids > unscored candidates > GitHub enrichment > company intel > KB maintenance

    Task types (in execution order):

    1. Company raids (priority 0)
    2. Blast through unscanned AI companies (LinkedIn profiles via Apify)
    3. ~50 companies per cycle, ~30s each

    4. When exhausted, triggers company replenishment

    5. Score unscoreds (priority 1)

    6. Find candidates evaluated today with score = 0

    7. Run PRISM scoring to populate scores

    8. GitHub enrichment (priority 2)

    9. High-scoring candidates (3+) with missing profile_data

    10. Query GitHub API for activity, repos, contributions

    11. Company intelligence (priority 3)

    12. Distinct companies from recent candidate evals (7-day window)

    13. Build company profiles for market research

    14. KB maintenance (priority 4)

    15. Re-index ChromaDB for stale embeddings
    16. Prune duplicate knowledge entries

    Key methods: - run_cycle() — Main overnight loop, checks VPS instance, cost cap, resets nightly - _pick_task() — Query DB for eligible work, rank by priority, return next task - _execute_task(task) — Route to appropriate module (raider, enricher, etc.) - get_last_night_summary() — Return summary of completed work

    State persistence: - data/overnight_state.json — Tracks nightly cost, completed tasks, timestamps - data/overnight.log — Logs all cycles, task selection, cost tracking

    2. Autonomy Loop (core/autonomy_loop.py)¶

    Self-improvement engine — FORGE detects bugs/gaps and fixes itself via a 3-model pipeline.

    Model Role Cost
    Haiku Research & root cause analysis ~$0.01–0.02 per cycle
    Opus Architectural design & risk assessment ~$0.10–0.20 per cycle
    Sonnet Implementation via copilot Copilot cost (tool_use)

    5-phase workflow:

    1. ANALYZE (Local)
    2. Read friction signals from data/friction.jsonl
    3. Mine failure logs from forge.db (errors, timeouts)

    4. Identify capability gaps from chat history

    5. RESEARCH (Haiku, cheap)

    6. Investigate root cause
    7. Read relevant source files (up to 100 lines each)

    8. Suggest approach

    9. DESIGN (Opus, deep)

    10. Review Haiku's findings
    11. Propose architectural changes
    12. Score confidence (0.0–1.0)

    13. Skip if confidence < 0.4

    14. EXECUTE (Sonnet + Copilot)

    15. Follow Opus's plan
    16. Write code, run tests

    17. Commit with detailed message

    18. TRACK

    19. Log result to autonomy_log.jsonl
    20. Update autonomy_state.json (improvements count, cost, last_cycle)

    Budgets: - MAX_IMPROVEMENTS_PER_DAY = 5 (env: AUTONOMY_MAX_DAILY) - DAILY_BUDGET_USD = 5.0 (env: AUTONOMY_DAILY_BUDGET) - CYCLE_INTERVAL_HOURS = 4 (env: AUTONOMY_CYCLE_HOURS)

    Runs: Every 4–6 hours, async loop in bot.py

    Dataclasses: - ImprovementCandidate — source, category, title, description, priority_score, evidence, estimated_complexity - ResearchReport — issue, root_cause, relevant_files, code_context, suggested_approach, cost_usd - DesignPlan — files_to_modify, changes_summary, risks, rollback_plan, confidence, cost_usd - ImprovementResult — candidate_title, status (success/failed/skipped/deferred), copilot_response, timestamp, cost_usd

    3. Dream Mode (core/dream_mode.py)¶

    Subconscious synthesis engine — connects unrelated knowledge, surfaces unexpected insights.

    Aspect Details
    When Midnight–6 AM Pacific (off-hours)
    Frequency Max 3 dreams per night
    Cost cap $0.50/night
    Model Haiku (fast, cheap)

    Process: 1. Gather context fragments: - Recent meeting action items - Recent candidates scored (5 most recent) - User facts (5 most recent) - Goals + milestones from KB

    1. Pass to _DREAM_SYSTEM prompt:
    2. Look for non-obvious patterns across domains
    3. Find skills/people from one area helping another
    4. Detect timing coincidences

    5. Flag contradictions needing resolution

    6. Synthesize → DreamInsight:

    7. title, synthesis (2–3 sentences), connections (pairs), confidence (0.0–1.0)

    8. Only store if confidence ≥ 0.5

    9. Store in dream_insights table:

    10. title, synthesis, connections (JSON array), confidence, dreamed_at, cost_usd, surfaced (flag)

    Dataclass: 

    @dataclass(frozen=True)
class DreamInsight:
    title: str
    synthesis: str
    connections: tuple  # ("fact A", "fact B"), ...
    confidence: float
    dreamed_at: str  # ISO timestamp
    cost_usd: float

    Log: data/dream_log.jsonl

    4. Dream Candidates (core/dream_candidates.py)¶

    24/7 autonomous channel discovery — finds new talent communities beyond explicitly configured channels.

    Feature Details
    Discovery methods AI-guided search for new platforms, communities, events
    Target stacks ClickHouse, OLAP, data engineering, Rust, Go, distributed systems, real-time analytics, observability
    Known platforms GitHub, LinkedIn, StackOverflow, Ashby (skip to avoid duplication)
    Cost estimation Ranks discoveries by relevance + cost impact
    Auto-wiring Approved channels auto-integrate into sourcing pipeline

    Dataclass: 

    @dataclass(frozen=True)
class ChannelDiscovery:
    id: int
    platform: str  # "discord", "slack", "meetup", "github_org", etc.
    name: str
    url: str
    description: str
    estimated_reach: int  # members/followers
    estimated_cost: str  # "free", "$X/mo", "one-time $X"
    relevance: float  # 0.0–1.0
    status: str  # "pending", "approved", "rejected", "wired"
    discovered_at: str

    Workflow: 1. Query channel_discoveries table for pending recommendations 2. Score by relevance × cost-benefit 3. Surface top N to Zack (via Discord or briefing) 4. Auto-wire approved channels into sourcing pipeline

    Scheduling Infrastructure¶

    APScheduler Core (modules/scheduler.py)¶

    Central task scheduler using APScheduler (asyncio).

    Component Details
    Timezone America/Los_Angeles (PST/PDT)
    Persistence SQLite scheduled_tasks table
    Task types Reminder (one-time), Recurring (cron), Special (briefing, raiding)
    Job defaults misfire_grace_time=3600, coalesce=True (skip missed occurrences)

    Initialization: 

    async def init_scheduler(bot, brain):
    # Load all enabled tasks from DB
    # Register each as APScheduler job (DateTrigger or CronTrigger)
    # Auto-clear browser cookies hourly
    # Catch up on missed recurring jobs (e.g., bot restart during scheduled task)

    DB Schema (scheduled_tasks): - id, description, task_type, cron_expression, run_at (ISO), enabled, created_at, last_run

    Key behaviors: - Cleanup past reminders — Disable one-time tasks with run_at in the past - Catch-up missed jobs — If bot restarts during a scheduled task, run it immediately - Auto-cookie-clear — Every hour, clear cookies from paywalled sites (bypass paywall limits)

    Task execution: - Defers to _execute_task(task) per task_type - Checks is_my_scheduler_window() — only this FORGE instance runs tasks (prevents duplicates) - Logs result to DB (last_run timestamp)

    Predictive Scheduling (core/predictive_scheduling.py)¶

    Learn user behavior patterns, schedule tasks intelligently.

    Component Details
    Input Message timestamps from chat history (mining when user is active)
    Pattern detection Day-of-week, hour-of-day histograms, frequency, consistency score
    Output Optimal timing for reminders, deferred tasks during busy windows
    Tables user_action_log, detected_patterns, suggestion_responses

    Pattern types: - Recurring behaviors (e.g., "Zack always checks messages 8–10 AM") - Busy windows (suppress reminders, defer expensive tasks) - Response patterns (when does Zack respond fastest to specific message types?)

    Key method: 

    async def should_send_now(
    importance: str,  # "low", "medium", "high"
    module: str       # "sourcing", "meeting", "maintenance"
) -> bool:

    Checks current hour against detected patterns — returns True if timing is good.

    Sleep Worker (modules/sleep_worker.py)¶

    Runs maintenance tasks when Discord channel is idle (> 120 seconds).

    Component Details
    Idle threshold 120 seconds without incoming messages
    Check interval Every 30 seconds
    Once-per-idle rule Runs once, then waits for new activity before re-running
    Tasks Memory decay, KB re-indexing, cache cleanup

    Usage: 

    # In on_ready()
from modules.sleep_worker import start_sleep_worker, note_activity

await start_sleep_worker(brain, bot, scheduler)

# In on_message()
note_activity()  # Reset idle timer

    Tasks executed (sequential): 1. Memory decay — reduce confidence on stale memories 2. KB cleanup — prune old, low-confidence entries 3. Cache eviction — clear old cached API responses 4. (More can be added)

    Company Sourcing Systems¶

    Company Raider (modules/company_raider.py)¶

    End-to-end company roster scraping — blast through target companies, discover candidates.

    Stage Details
    Input Company name (e.g., "Resolve AI")
    Search Apify LinkedIn actor (batch: 3–5yr + 6–10yr experience)
    Filter Confirm match, 2–7yr experience, Bay Area location
    Dedup Against Ashby pipeline, candidate_evals, profile_scrape_cache
    Store ALL profiles (8+ yr → senior_backlog tag)
    Output Qualified candidates ready for PRISM scoring

    Key constraint: - ALWAYS batch Apify calls — 2 calls per company (3–5yr, 6–10yr), never 1 profile at a time - Previous violations cost $200+ in wasted Apify credits

    Engineering keywords: engineer, developer, MTS, ML, AI, software, SWE, backend, full stack, data scientist, etc.

    Bay Area filter: 50+ US cities + "Bay Area", "SF Bay", "San Francisco Bay", etc.

    Blocklist: HockeyStack, Stripe, big tech (Google, Meta, Apple, Microsoft, etc.)

    Usage: 

    from modules.company_raider import raid, print_raid_results

results = await raid("Resolve AI", max_profiles=50)
print(print_raid_results(results))

    Company Replenisher (modules/company_replenisher.py)¶

    Auto-discover new companies to raid — mine previous employers, search web for new AI startups.

    Strategy Source Frequency
    Candidate mining Extract companies from profiles scored ≥ 3.0 Every cycle
    Web discovery Search YC batches, funding announcements, AI startup rankings Every cycle or weekly

    Data source: data/ai_companies_master.csv

    Blocklist (→ never add): - HockeyStack, all big tech (Google, Meta, Amazon, Microsoft, Apple, etc.) - Big finance & consulting (JPMorgan, Goldman Sachs, Deloitte, etc.) - Big enterprise (Salesforce, Adobe, VMware, SAP, etc.) - Research labs (UC Berkeley, Stanford, MIT, etc.) - Junk artifacts (Stealth Mode, Full-Time, Self-Employed, etc.)

    Usage: 

    from modules.company_replenisher import replenish_from_candidates, replenish_from_web

added = await replenish_from_candidates(min_score=3.0)  # Mine DB
added = await replenish_from_web()  # Web search

    News Aggregation (modules/news.py)¶

    RSS-based news feed — polls 20+ sources every 30 minutes, posts to 📰-news channel.

    Source Type Frequency
    Washington Post (5 feeds), Reuters, AP, CNN, BBC, Guardian, NPR, Intercept, ProPublica, Wired, Ars Technica Mainstream + investigative Every 30 min

    Deduplication: SQLite seen_items table (7-day retention)

    Rich embeds: Category emoji, source color coding, article summary (Claude Haiku)

    Max posts per poll: 5 (avoid flooding)

    Cost Model¶

    Overnight Engine ($10/night cap)¶

    Task Typical cost Frequency
    Company raid (50 companies) $1.50–2.50 Every cycle
    Score unscoreds (batch of 10) $0.10–0.20 1× per cycle
    GitHub enrichment (batch of 5) $0.05–0.10 1× per cycle
    Company intel research $0.50–1.00 1× per cycle
    KB maintenance <$0.01 1× per cycle
    Total per cycle $2–4 4 cycles/night
    Total per night $8–16 Capped at $10

    Autonomy Loop ($5/day cap)¶

    Phase Model Typical cost
    Research Haiku (100 lines code, ~500 tokens) $0.01–0.02
    Design Opus (architecture review, 1000+ tokens) $0.10–0.20
    Execute Sonnet + Copilot (tool_use x 10 iter) $0.50–1.00
    Per improvement — $0.61–1.22
    5 improvements/day — $3.05–6.10

    Dream Mode ($0.50/night cap)¶

    Task Cost
    Gather context (DB queries) <$0.001
    Haiku synthesis (3 dreams max) $0.10–0.15 per dream
    3 dreams/night $0.30–0.45

    Monitoring & Debugging¶

    View overnight progress¶

    # Check last night's summary
sqlite3 data/forge.db "SELECT * FROM overnight_log ORDER BY created_at DESC LIMIT 5;"

# Tail overnight engine logs
tail -f data/overnight.log | grep "Overnight"

    View autonomy improvements¶

    # Check improvement history
cat data/autonomy_log.jsonl | jq '.[] | {title, status, cost_usd}'

# Check current state
cat data/autonomy_state.json | jq .

    View dream insights¶

    # Check dream log
sqlite3 data/forge.db "SELECT title, confidence, dreamed_at FROM dream_insights ORDER BY dreamed_at DESC LIMIT 5;"

    Disable/enable systems¶

    # Disable overnight engine (set instance to "local")
export FORGE_INSTANCE=local

# Disable autonomy (set daily budget to 0)
export AUTONOMY_DAILY_BUDGET=0

# Disable dream mode (set cost cap to 0)
export DREAM_COST_CAP=0
