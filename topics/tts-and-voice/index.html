    Text-to-Speech & Voice System

    9 min read · Last updated April 04, 2026 · 1,845 words

    FORGE integrates ElevenLabs AI voice synthesis for reading text, messages, URLs, and morning briefs aloud. The system features intelligent voice rotation that ensures you never hear the same voice twice in a row, with audio files stored locally for caching and replay.

    Overview

    The TTS system consists of three layers: 1. API layer (modules/tts.py) — ElevenLabs integration, voice rotation logic, MP3 generation 2. Action handlers (cogs/tts_actions.py) — Discord command dispatch for read text/last/URL/brief 3. Trigger intercept (bot.py on_message) — Fast-path recognition of "read this" phrases before classifier

    Architecture

    Message → on_message TTS intercept
         ↓
    "read this ..." → tts_read_text
    "read that" → tts_read_last
    "read brief" → tts_read_brief
         ↓
    classifier (if no TTS intercept match)
         ↓
    brain / cog dispatch

    The TTS intercept is positioned at the top of on_message (before classifier) to provide sub-100ms response for common voice commands.

    ElevenLabs Integration

    Provider: ElevenLabs API v1
    Base URL: https://api.elevenlabs.io/v1
    Auth: ELEVENLABS_API_KEY in .env
    Model: eleven_turbo_v2_5 (fast + cost-efficient, default)
    Quality model: eleven_multilingual_v2 (highest quality, 29 languages)

    API Flow

    POST /v1/text-to-speech/{voice_id}
Headers:
  - xi-api-key: <ELEVENLABS_API_KEY>
  - Content-Type: application/json
Body:
  {
    "text": "...",
    "model_id": "eleven_turbo_v2_5",
    "voice_settings": {
      "stability": 0.5,      # 0 = expressive, 1 = monotone
      "similarity_boost": 0.75
    }
  }

Response:
  200 OK → audio/mpeg binary stream (MP3)
  != 200 → JSON error message

    Voice Rotation System

    Never Repeat Last Voice

    The voice selection logic guarantees you never hear the same voice twice in a row. This is critical for morning briefs, daily reads, and repeated use cases.

    Algorithm (in pick_daily_voice()):

    1. Load voice usage history from data/tts/voice_history.json
    2. Filter available voices: exclude used list AND last_voice
    3. If all voices exhausted: reset used list (but still exclude last_voice)
    4. Pick random voice from filtered pool
    5. Append voice_id to used list and save history
    6. Log: "Today's voice: {name} ({accent}, {gender})"

    Voice History Schema

    {
  "used": ["voice_id_1", "voice_id_2", "..."],
  "last_voice": "voice_id_3",
  "last_date": "2026-04-04"
}

    Storage: data/tts/voice_history.json

    Voice Metadata

    Each voice from GET /v1/voices includes: - voice_id: Unique identifier (used in TTS requests) - name: Display name (e.g., "Adam", "Alice", "Luna") - preview_url: Sample audio clip - labels: Object with accent (e.g., "American"), gender (e.g., "male")

    Text-to-Speech API

    Module: modules/tts.py

    Main Function: text_to_speech()

    async def text_to_speech(
    text: str,
    output_path: str | Path | None = None,
    voice_id: str | None = None,
    model: str = DEFAULT_MODEL,
    stability: float = 0.5,
    similarity_boost: float = 0.75,
) -> Path:
    """Convert text to speech using ElevenLabs.

    Args:
        text: Text to speak
        output_path: Where to save MP3. Auto-generated if None.
        voice_id: Specific voice to use. If None, picks today's rotation voice.
        model: ElevenLabs model ID
        stability: Voice stability (0-1). Lower = more expressive.
        similarity_boost: How closely to match the voice (0-1).

    Returns:
        Path to generated MP3 file.
    """

    Behavior: - If voice_id is None: calls pick_daily_voice() to get rotation voice - Generates timestamped filename: brief_20260404_145230.mp3 - Saves to data/tts/ directory - Logs file size (KB), character count, and voice name - Raises ValueError if API fails or no API key configured

    Example Usage:

    # Auto voice rotation
audio_path = await text_to_speech("Hello, this is a test.")
# Returns: Path("data/tts/brief_20260404_145230.mp3")

# Specific voice
audio_path = await text_to_speech(
    "Hello, this is a test.",
    voice_id="adam",  # Use "Adam" voice specifically
    stability=0.3,     # More expressive
)

    Helper Functions

    Function Purpose
    _get_api_key() Loads ELEVENLABS_API_KEY from .env or environment
    _load_voice_history() Reads voice_history.json, returns dict with used/last_voice
    _save_voice_history(history) Writes voice history to JSON
    get_available_voices() Fetches full voice list from ElevenLabs API
    pick_daily_voice(voices) Selects next voice in rotation, never repeats last
    speak_alert(text, channel) Generate + post audio alert to Discord channel
    morning_brief_audio(brief_text) Render morning brief as audio (see morning-brief)

    Action Handlers

    Module: cogs/tts_actions.py
    Cog: TTSCog

    Dispatched by classifier when TTS actions detected. Each handler runs async, posts status/result to Discord.

    tts_read_text — Read Arbitrary Text

    Trigger: "read this text", "say this message", "read aloud: ..."
    Handler: _read_text(intent, message)

    1. Extract text from message or intent.query
    2. Try prefixes: "read this:", "say this:", "read aloud:", etc.
    3. Call text_to_speech(text)
    4. If MP3 < 10 MB: post as Discord audio file
    5. If >= 10 MB: post error message suggesting shorter text

    Prefixes Recognized: - "read this out loud:" / "read this out loud" - "read this:" / "read this" - "say this:" / "say this" - "read aloud:" / "read aloud" - "speak:" / "speak"

    tts_read_last — Read Previous Message

    Trigger: "read that", "read the last message", "read that out loud"
    Handler: _read_last(message)

    1. Search channel history (limit 5) for previous non-bot message
    2. Fallback to bot messages if no human message found
    3. Call text_to_speech(previous_message.content)
    4. Post MP3 to channel

    Error Handling: Returns "No previous message to read." if history is empty.

    tts_read_url — Scrape, Summarize, Read URL

    Trigger: "read this URL", "read this link"
    Handler: _read_url(intent, message)

    1. Extract URL from intent.query or message content (regex)
    2. Fetch page via aiohttp (timeout: 15s)
    3. Strip HTML tags with regex: <[^>]+> → space
    4. Collapse whitespace
    5. Truncate to first 3000 chars (avoid ElevenLabs char limits)
    6. Call text_to_speech(text)
    7. Post MP3 to channel

    Error Handling: - URL fetch timeout or non-200 status → "Couldn't fetch URL (status X)" - TTS API error → propagated to user

    tts_read_brief — Morning Brief Audio

    Trigger: "read brief", "read the brief", "listen to brief"
    Handler: _read_brief(message)

    1. Check for local file: data/tts/latest_brief.md
    2. If not found: search channel history (limit 20) for "Morning Brief" or "Dream Cycle" text
    3. Call morning_brief_audio(brief_text) to render audio
    4. Post MP3 with caption "Morning Brief — audio:"

    Related: See morning-brief for brief generation pipeline.

    TTS Intercept (Fast-Path)

    Location: bot.py on_message() — positioned before classifier

    Trigger Recognition:

    # Normalize punctuation after trigger phrase
_tts_normalized = re.sub(
    r"^(read this|say this|speak this|forge read|forge say|tts|read aloud)[.,;:!?\s]+",
    r"\1 ",
    message.content.lower()
)

# Check if starts with normalized trigger
if _tts_normalized.startswith(("read this ", "say this ", "tts ", ...)):
    action = "tts_read_text"
    query = extracted_text
    → dispatch to TTSCog immediately

    Phrases Matched: - "read this. text" → "read this text" (punctuation normalized) - "read this, text" → "read this text" - "read this: text" → normalized to "read this text" - "say this! text" → normalized - "tts text" → direct pass-through - "read that", "read the last message" → tts_read_last - "read brief", "listen to brief" → tts_read_brief

    Why Pre-Classifier? - Eliminates 50-100ms of classifier latency for common voice use cases - Ensures "read this" feels instant (Discord UX) - Reduces API token spend (no brain call for trivial reads)

    data/tts/ Output directory for generated MP3 files —
    data/tts/voice_history.json Voice rotation state (persisted) —
    data/tts/latest_brief.md Morning brief text (used by tts_read_brief) —

    Configuration

    Environment Variables: - ELEVENLABS_API_KEY — Required. Loaded from .env via _get_api_key()

    Model Defaults: - DEFAULT_MODEL = "eleven_turbo_v2_5" (fast + cost-efficient) - Can override per call: text_to_speech(..., model="eleven_multilingual_v2")

    Voice Settings: - stability=0.5 — Default (balanced) - similarity_boost=0.75 — Default (strong voice matching) - Customizable per call

    Storage: - Voice history: data/tts/voice_history.json - MP3 output: data/tts/brief_<timestamp>.mp3 - Directory auto-created if missing

    Integration with Other Systems

    Morning Brief

    morning-brief calls morning_brief_audio() to render the daily briefing as audio. Results in scheduled podcast posted to Discord.

    Flow: 1. Autonomy loop generates brief text → data/tts/latest_brief.md 2. morning_brief_audio(brief_text) converts to MP3 3. Brief is posted to Discord with audio attachment 4. Users can say "read brief" to replay

    Meeting Summaries

    TTS can render meeting summaries to audio for async listening. Not yet integrated into scheduled flows — see future-work for roadmap.

    Alert Sounds

    speak_alert(text, channel) generates and posts audio alerts for: - High-priority candidate drops - Crash notifications - Important FORGE state changes

    Example: 

    await speak_alert("New candidate matched: Alice Chen", channel)

    Voice Gallery

    ElevenLabs offers 30+ voices across accents (American, British, Indian, etc.) and genders. Full list fetched dynamically via get_available_voices().

    Common Voices: - Adam (American male) - Alice (American female) - Luna (British female, distinctive) - Sophia (American female, warm) - Jackson (American male, deep) - Rachel (American female, professional)

    Run /current-voice or check logs to see today's selected voice.

    Performance & Limits

    Metric Limit Notes
    Max text length ~5000 chars ElevenLabs API limit. TTS errors if exceeded.
    Output MP3 size < 10 MB Hard Discord file limit. Cog warns if exceeded.
    API rate ~100 req/min Based on subscription tier. No explicit rate limiting in code.
    Voice list cache Per request Fetched fresh from ElevenLabs every text_to_speech() call. Consider caching if high volume.
    File generation ~2-5s Depends on text length and network latency.

    Error Handling

    Error Handling Recovery
    Missing ELEVENLABS_API_KEY Raises ValueError Add API key to .env, restart bot
    ElevenLabs API returns non-200 Raises ValueError with HTTP status + error text Check API key, quota, network
    No voices available from API Raises ValueError Temporary API issue; retry
    MP3 > 10 MB User gets warning message Ask user to provide shorter text
    URL fetch timeout (15s) User gets "Couldn't fetch URL" URL may be slow or unresponsive
    Channel history empty (read_last) User gets "No previous message to read." Need at least one previous message
