    Meeting Analysis: HockeyStack Team Sync w/ Arda & Hank

    🔑 TL;DR

    The team is refining their sourcing strategy for a ClickHouse database engineer role by widening the search net to include anyone with ClickHouse keywords (removing strict title filters), excluding only managers and site reliability engineers, while avoiding Altinity but including ClickHouse company candidates.

    📋 Action Items

    Zack - Remove strict job title filters and search primarily by ClickHouse keyword on LinkedIn Recruiter - Add software engineer to title searches when using title filters - Exclude manager and site reliability engineer titles from searches - Remove Altinity candidates from sourcing list - Include ClickHouse company employees in candidate pool - Source candidates from Netherlands/Amsterdam area from GitHub scrape - Provide list of Altinity-like companies to Arda and Hank - Re-run searches on Series A-C companies using broader ClickHouse keyword approach - Implement the LinkedIn scraping API that Arda provided - Ask questions in the DM group if stuck on API implementation

    Hank - Review and finalize job description for release next week

    Unassigned - None

    ✅ Decisions Made

    • Widen the sourcing net by removing strict title constraints and focusing on ClickHouse as a keyword
    • Exclude managers and site reliability engineers from searches
    • Accept candidates with 1+ years of ClickHouse experience if they worked at scale
    • Accept candidates who use multiple databases alongside ClickHouse (e.g., Kubernetes + MySQL/Postgres)
    • Do not source from Altinity, but DO source from ClickHouse company itself
    • Include candidates from Netherlands/Amsterdam in sourcing efforts
    • Use exclusion-based filtering (removing unwanted titles) rather than inclusion-based filtering
    • Rely on job description and self-selection to help filter candidates once outreach begins
    • Leverage automation (LinkedIn API) rather than manual work where possible

    💬 Talking Points

    • Zack has sourced 35-40 candidates so far, finding it challenging due to the rarity of 5+ years ClickHouse experience and unclear job titles
    • Most GitHub repository contributors are Russia-based, limiting the viable candidate pool from that source
    • LinkedIn Recruiter profiles show more detail than standard profiles, making it difficult for Arda/Hank to see what Zack sees during calibration
    • The team discussed whether candidates need ClickHouse exclusively vs. experience with adjacent technologies (Kafka, Postgres, MySQL, OLAP/OLTP databases)
    • Max (team member) was used as an example of someone who has ClickHouse skills but doesn't elaborate on it in their LinkedIn profile, highlighting the need for broader keyword searches
