    Company Raider

    End-to-end company sourcing pipeline — LinkedIn search, deduplication, and talent extraction.

    File: modules/company_raider.py | Lines: ~449

    What It Does

    Raids a company for engineering talent. Batches LinkedIn profile searches (2 calls per company: 3-5yr and 6-10yr experience) via RapidAPI, post-filters by location (Bay Area) and experience, deduplicates against Ashby pipeline and candidate cache, and returns qualified candidates ready for PRISM scoring. Stores all profiles with senior_backlog tag for profiles 8+ years.

    Key Functions

    Function Purpose
    raid(company_name, max_profiles) Main entry point — batch search, dedupe, return results
    _search_linkedin(query, count) RapidAPI call to search for profiles
    _dedupe_against_pipeline(profiles) Filter profiles already in Ashby
    _filter_by_location(profiles) Keep Bay Area only
    _filter_by_experience(profiles) Keep 2-7yr (or tag 8+ for backlog)
    print_raid_results(results) Format for user display

    Dependencies

    • aiohttp — async HTTP for RapidAPI
    • RapidAPI: fresh-linkedin-profile-data.p.rapidapi.com
    • Local: Ashby dedupe, candidate cache dedup
    • Bay Area location set (58 cities)
